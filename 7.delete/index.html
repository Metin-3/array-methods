delete Arry Method

delete operatoru JavaScript-də obyektlərdən və ya massivlərdən müəyyən bir xüsusiyyəti (property) və ya elementi silmək üçün istifadə olunur. Bu operator verilmiş xüsusiyyətin tamamilə obyektin tərkibindən çıxarılmasını təmin edir.

Əsas Xüsusiyyətlər

  1. Xüsusiyyətləri silir: Obyektlərin daxilindəki xüsusiyyətləri tamamilə çıxarır.
  2. İndeksləri tərpətmir: Massivlərdə bir elementi sildikdə indeksləri yenidən düzənləmir. Bunun əvəzinə həmin indeksdə undefined qalır.
  3. Daha çox obyektlər üçün uyğundur: Massivlərdə elementləri silmək üçün delete yerinə splice() daha məqsədəuyğundur.
  4. Mutasiya edir: Obyektin orijinal strukturu dəyişir.

Qeyd ediləsi məqamlar

delete yalnız obyektin öz xüsusiyyətlərini silir. Prototipdə olan xüsusiyyətlərə təsir etmir. JavaScript-də obyektlər prototiplər əsasında qurulur. Hər bir obyektin bir prototipi ola bilər və bu prototip vasitəsilə obyekt başqa bir obyektin xüsusiyyətlərini (property) və metodlarını irsi olaraq əldə edə bilər. Misal ilə izah function Person(name) {
this.name = name; // Bu obyektin öz xüsusiyyətidir.
}
// Prototipdə bir xüsusiyyət əlavə edirik
Person.prototype.age = 25;
let john = new Person("John");
// john obyektində olan xüsusiyyətlər:
console.log(john.name); // "John" (john obyektinin özünə aiddir)
console.log(john.age); // 25 (john obyektinin prototipindən gəlir)
 delete nə edir? 1--> Obyektin öz xüsusiyyətlərini silə bilir: delete john.name; // john obyektindəki `name` xüsusiyyəti silinir.
console.log(john.name); // undefined
 2--> Prototipdən gələn xüsusiyyətləri silməz: delete john.age; // Bu prototipdə olduğuna görə təsir etmir.
console.log(john.age); // 25 (prototipdən gəlməyə davam edir)
 Niyə belə olur? john obyektinin özündə age xüsusiyyəti yoxdur. Bu xüsusiyyət onun prototipindən gəlir. delete yalnız obyektin özündə olan xüsusiyyətləri silə bilər. Prototipdəki xüsusiyyətlər dəyişməz qalır. Prototipdə olan xüsusiyyətləri necə silmək olar? Əgər prototipdən gələn bir xüsusiyyəti silmək istəyirsinizsə, onu obyektin prototipindən birbaşa silməlisiniz: delete Person.prototype.age; // Prototipdən `age` xüsusiyyətini silirik.
console.log(john.age); // undefined
 Bu fərqi başa düşmək niyə vacibdir? Öz xüsusiyyətlər (Own Properties): Birbaşa obyektin tərkibində olan xüsusiyyətlərdir Prototipdən gələn xüsusiyyətlər (Inherited Properties): Prototip zəncirindən gələn xüsusiyyətlərdir.

delete yalnız obyektin öz xüsusiyyətlərini silir. Əgər obyektin daxilində olmayan, amma prototipdən gələn bir xüsusiyyəti silmək istəyirsinizsə, bunu prototip səviyyəsində etmək lazımdır

Əgər xüsusiyyətin configurable atributu false-dirsə, onu delete ilə silmək mümkün deyil. JavaScript-də obyekt xüsusiyyətlərinin (property) müəyyən atributları var və bu atributlar xüsusiyyətin davranışını tənzimləyir. Bu atributlardan biri də configurable atributudur. Əgər bir xüsusiyyətin configurable atributu false-dirsə, bu xüsusiyyət müəyyən məhdudiyyətlərə malik olur, məsələn:

1--> Bu xüsusiyyəti delete ilə silmək mümkün deyil.

2--> Xüsusiyyətin atributlarını dəyişdirmək mümkün olmur (məsələn, writable, enumerable, configurable kimi atributlar dəyişdirilə bilmir).

configurable atributu nədir? configurable: true — Xüsusiyyət silinə bilər və atributları dəyişdirilə bilər. configurable: false — Xüsusiyyət sabitdir və silinə bilməz. const obj = {};
// Yeni bir xüsusiyyət əlavə edirik və onun `configurable` atributunu `false` təyin edirik:
Object.defineProperty(obj, "name", {
value: "John",
configurable: false, // Bu xüsusiyyət sabitdir və silinə bilməz.
writable: true, // Dəyərlər yenə də dəyişdirilə bilər.
enumerable: true // Dövr edən metodlarda görünür.
});
console.log(obj.name); // "John"
// `delete` ilə bu xüsusiyyəti silməyə çalışaq:
delete obj.name;
console.log(obj.name); // "John" (xüsusiyyət hələ də mövcuddur!)
Burada, delete işləmədi, çünki name xüsusiyyətinin configurable atributu false təyin edilmişdir. Niyə configurable false təyin edilir?

configurable: false bir xüsusiyyətin sabit olmasını təmin edir və onun strukturu üzərində dəyişiklik edilməsinin qarşısını alır. Bu, xüsusilə aşağıdakı vəziyyətlərdə faydalıdır:

1--> Əsas məlumatların qorunması üçün: Əgər bir obyektin müəyyən xüsusiyyətlərinin dəyişdirilməməsini istəyirsinizsə, configurable: false istifadə edərək onları qoruyursunuz. 2--> Daha təhlükəsiz API-lər yaratmaq üçün: Böyük tətbiqlərdə təsadüfi silinmə və ya dəyişikliklərin qarşısını almaq üçün. configurable atributunun dəyişdirilməzliyi Əgər bir xüsusiyyətin configurable atributu bir dəfə false təyin edilibsə, onu sonradan dəyişdirmək mümkün deyil. Object.defineProperty(obj, "name", {
configurable: true // Bu xətaya səbəb olacaq!
});
 configurable: false ilə digər atributlar arasındakı əlaqə Dəyər dəyişdirilə bilər (yazıla bilər), əgər writable: true olarsa. obj.name = "Doe"; // Bu işləyəcək, çünki writable: true
console.log(obj.name); // "Doe"
Atributları dəyişdirmək mümkün deyil: Object.defineProperty(obj, "name", {
writable: false // Bu xəta verəcək!
});

configurable atributu obyekt xüsusiyyətlərinin necə işləyəcəyini idarə edir:
true olduqda, xüsusiyyət silinə və atributları dəyişdirilə bilər.
false olduqda, xüsusiyyət sabit qalır və silinə bilməz.
Bu, obyektlərə daha çox nəzarət etməyə imkan verir və mühüm xüsusiyyətlərin təhlükəsizliyini təmin edir.

Geri qaytarır:

true : Əgər xüsusiyyət uğurla silinibsə.

false : Əgər xüsusiyyəti silmək mümkün deyilsə.

Performans Qeydləri

delete operatorunun performansı yaxşıdır, lakin çox istifadə edildikdə yavaşlıq yarana bilər.

Massivlərdə elementləri silmək üçün splice() istifadə etmək daha məqsədəuyğundur.

delete operatoru obyektlərdən və ya massivlərdən xüsusi elementləri silmək üçün sadə və güclü bir vasitədir. Obyektlərlə işləyərkən istifadə üçün idealdır, lakin massivlərdə istifadə edildikdə ehtiyatlı olmaq və mümkün olduqda splice() kimi metodlardan istifadə etmək daha məqsədəuyğundur.
